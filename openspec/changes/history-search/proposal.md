## Why

用户在历史记录面板中查找特定内容时，需要逐条滚动浏览，效率低下。当历史记录积累到数百条时，快速定位目标内容成为痛点。需要一个即时搜索功能，让用户输入关键词就能快速过滤，类似 vim 的 `/` 搜索模式。

## What Changes

- 新增搜索模式：用户按 `/` 键进入搜索模式，ESC 退出
- 实时过滤：输入字符即时匹配历史记录内容，无需回车确认
- 模式隔离：在搜索模式中，`/` 作为普通字符输入（不触发搜索退出）
- 非阻塞：过滤过程为异步进行，不中断用户连续输入
- 保留原有交互：搜索模式不影响历史记录的复制、删除等操作

## Capabilities

### New Capabilities
- `history-search`: 历史记录面板的即时搜索与过滤功能，包括搜索模式管理、实时匹配、键盘导航

### Modified Capabilities
（无）

## Impact

**受影响代码：**
- 前端历史记录面板（`src/` 下的 JS/HTML/CSS）
- 键盘事件处理逻辑
- 历史记录数据展示逻辑

**依赖影响：**
- 无新增外部依赖，使用原生 JS 实现

**系统影响：**
- 不影响后端 Rust 逻辑
- 不影响 WebSocket 通信
- 不影响数据存储
