## Context

PaseBoard 是一个 Tauri v2 桌面应用（Rust 后端 + 原生 JS 前端），窗口固定 460x720px。当前 UI 有三个标签页（设备列表、历史记录、控制台），设置入口是状态栏右下角的 ⚙️ 下拉菜单，仅包含"开发者模式"开关。

本次改动将下拉菜单替换为设置弹窗（Modal），新增列显隐控制功能。

## Goals / Non-Goals

**Goals:**
- 用户可通过设置弹窗控制设备列表和历史记录表格的列显隐
- 设置持久化到 localStorage，重启后保持
- 弹窗复用配对弹窗的深色主题，视觉统一
- 开发者模式开关迁入弹窗，功能不变
- Ctrl+Shift+D 快捷键在弹窗打开/关闭状态下均有效

**Non-Goals:**
- 不跨设备同步设置
- 不改后端代码
- 不引入前端测试框架
- 不做响应式布局适配

## Decisions

### 1. CSS data-col 属性选择器方案

给 `<th>` 和 `<td>` 添加 `data-col` 属性（如 `data-col="status"`），通过在 `<table>` 上切换 CSS class 配合属性选择器控制显隐。

**理由：** 不需要修改 `loadDevices()` / `loadHistory()` 的渲染逻辑，不需要数据缓存、hash 缓存重置、colspan 重算。改动量最小。

### 2. 深色主题视觉风格

复用配对弹窗的深色主题（`#1e1e2e` 背景，`#cdd6f4` 文字），使用独立 CSS 类名。

**理由：** 视觉统一，避免两套弹窗风格不一致。

### 3. localStorage 存储

使用 `paseboard:columnVisibility` key，JSON 对象格式。缺失 key 或解析失败时默认所有列可见。

**理由：** 与现有 `paseboard:devMode` 命名风格一致，向前兼容。

## Risks / Trade-offs

- **CSS 方案的局限性：** 隐藏列仍在 DOM 中，只是不可见。对性能无影响（列数极少），但如果未来需要"导出可见列"功能，需要额外处理。
- **深色弹窗 vs 浅色主界面：** 设置弹窗是深色，主界面是浅色。视觉对比明显，但作为"浮层"是合理的（配对弹窗也是深色浮层）。
