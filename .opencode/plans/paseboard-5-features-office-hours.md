## Problem Statement

PaseBoard 是一个跨平台局域网粘贴板同步工具，当前已完成：
- 文本同步（mDNS 发现 + WebSocket 传输）
- 安全加密架构（Ed25519 身份 + ECDH 密钥交换 + AES-GCM 传输加密 + SQLCipher 存储加密）
- 设备发现与连接管理
- 基础 UI（设备列表 + 历史记录）

五个待解决的需求：
1. **清空历史按钮** — 历史记录只能逐条删除，没有一键清空
2. **隐藏 macOS Dock 图标** — 菜单栏应用不需要 Dock 图标
3. **修复双菜单栏图标** — `tauri.conf.json` 和 `TrayIconBuilder` 各创建一个图标，macOS 上出现两个
4. **安全配对** — 当前任何局域网设备都可互连，缺少设备确认流程
5. **图片同步与预览** — 当前只支持文本，图片复制后不会同步

## What Makes This Cool

- 图片同步让 PaseBoard 从「文本粘贴板工具」变成「真正的粘贴板工具」——复制图片，另一台设备瞬间预览
- 安全配对完成后，加密架构真正落地——不再是"门没锁"的状态
- 三个小修复让日常使用体验干净利落

## Constraints

- 保持零配置体验，不能要求用户注册账号或配置服务器
- 必须向后兼容已有连接过的设备
- macOS 特定功能不能破坏 Windows/Linux 构建
- 所有改动遵守 CLAUDE.md 中的技术栈选择（Tauri v2（注：CLAUDE.md 写的是 v1 但实际 Cargo.toml 已是 v2）, Rust, arboard, SQLCipher, 原生 HTML/CSS/JS）

## Premises

1. PaseBoard 核心价值是"零配置的局域网粘贴板同步"，功能扩展不能破坏零配置体验
2. 功能顺序：清空历史 + Dock 隐藏 + 双图标修复 → 安全配对 → 图片同步
3. 图片同步全量实现（检测 → PNG 编码 → WebSocket 传输 → SQLite BLOB 存储 → 缩略图生成 → UI 预览 → 恢复到剪贴板）
4. 安全配对用信任首次连接模式（陌生设备弹确认 → 接受后永久记住）
5. 方向是"桌面信息流枢纽"（未来文件拖拽、历史搜索），但不超前设计

## Approaches Considered

### Approach A: 逐个推进（推荐）
每个功能独立 PR，按序逐一完成。不改架构，不改现有数据流。

### Approach B: 消息系统重构 → 加功能
先重构 Message 为通用 ClipboardContent 枚举，再往上加功能。

### Approach C: 两波发布
Wave 1: 小修复 + 配对，Wave 2: 图片同步。

## Recommended Approach

**Approach A: 逐个推进。** 原因是：
- Side project 节奏：每完成一个功能立刻能用
- 互不阻塞，出问题只影响单个功能
- 代码评审简单，适合单人开发

## Feature Specifications

### F1: 清空历史按钮（~0.5 天）

**后端：** `src-tauri/src/clipboard/storage.rs`
- 新增 `pub fn clear_all(&self) -> Result<()>` 方法，执行 `DELETE FROM clipboard_history`
- 新增 IPC 命令 `clear_history` 暴露给前端

**前端：** `ui/index.html`
- 历史记录区域添加一个「清空」按钮
- 点击后确认对话框（防止误操作）
- 确认后调用 `invoke('clear_history')`，刷新列表

**注意：** 清空后数据不可恢复。建议 `DELETE` 在事务内执行。暂不执行 `VACUUM`（SQLCipher 执行 VACUUM 有性能开销，且磁盘空间回收不是关键需求）。

**改动文件：** `storage.rs`, `lib.rs`, `index.html`

### F2: 隐藏 macOS Dock 图标（~0.3 天）

**方案：** 在 `main.rs` 的 `setup()` 回调中，条件编译调用 macOS 特定 API：

```rust
#[cfg(target_os = "macos")]
{
    use tauri::ActivationPolicy;
    app.handle().set_activation_policy(ActivationPolicy::Accessory)?;
}
```

**⚠ 待验证：** Tauri v2 的 `tray-icon` feature 是否暴露 `set_activation_policy` 或类似 API？如果 Tauri 层不提供，备选方案是通过 `objc2` crate 直接调用 `NSApp.setActivationPolicy(.NSApplicationActivationPolicyAccessory)`，但这需要破例使用 unsafe 代码（违反 CLAUDE.md 规则）。编码前必须先确认 Tauri API 的可行性。

**改动文件：** `lib.rs`, `Cargo.toml`（如需加 `objc2`）

### F3: 修复双菜单栏图标（~0.2 天）

**根因：** `tauri.conf.json` 中存在 `trayIcon` 配置块，同时 `main.rs` 中 `TrayIconBuilder::new()` 创建第二个图标。

**方案：** 删除 `tauri.conf.json` 中的 `trayIcon` 配置（`iconPath`、`iconAsTemplate` 等），只保留代码中的 `TrayIconBuilder`。

**改动文件：** `tauri.conf.json`

### F4: 安全配对（~2 天）

**数据结构：** `src-tauri/src/config.rs`
- `AppConfig` 新增字段 `paired_devices: Vec<PairedDevice>`（线程安全：通过 `Arc<RwLock<...>>` 或专用 channel 访问）
- `PairedDevice { device_id: String, device_name: String, public_key: String, paired_at: DateTime }`
- 新增 `auto_accept_pairing: bool` 默认 true（向后兼容）
- 新增 `remove_paired_device(device_id)` 和 `list_paired_devices()` 方法

**连接时序（关键）：**
1. WebSocket 连接建立后，**第一条消息必须是** `PairingRequest`
2. 发送方发完 `PairingRequest` 后，等待 `PairingAccept` / `PairingReject` / 超时（**30 秒超时**）
3. 超时未响应 → 断开连接，30 分钟后可重试
4. 接收方收到 `PairingRequest` 后连接进入 **pending 状态**，心跳保持活跃但不传输数据
5. 用户决定后 resolve 状态

**后端流程：**
1. 设备发现阶段：mDNS TXT 记录加 `pairing=paired` / `pairing=unpaired`
2. 已配对设备自动连接（走现有流程，跳过配对步骤）
3. 未配对设备连接时，发送 `PairingRequest { device_id, device_name, public_key }`
4. 接收方收到后：
   - 如果 `auto_accept_pairing == true` → 自动配对（向后兼容）
   - 否则 → 触发前端配对请求弹窗
5. 用户选择接受 → 发送 `PairingAccept { device_id, public_key }`，双方存入 config
6. 用户选择拒绝 → 发送 `PairingReject`，断开连接

**消息类型新增：** `Message::PairingRequest { device_id, device_name, public_key }`, `PairingAccept { device_id, public_key }`, `PairingReject`, `PairingTimeout`

**前端：** `ui/index.html`
- 配对请求弹窗（设备名、公钥指纹、接受/拒绝按钮）
- 设备列表显示配对状态（已配对/未配对）
- 已配对设备可点击"忘记设备"解除配对（调用 `remove_paired_device` IPC）

**改动文件：** `config.rs`, `message.rs`, `app.rs`, `mdns.rs`, `index.html`, `lib.rs`

### F5: 图片同步与预览（~3-4 天）

**剪贴板检测：** `src-tauri/src/clipboard/monitor.rs`
- 先检测文本（`get_text()`），无文本则尝试图片（`get_image()`）
- `ClipboardChange` 增加 `content_type: enum { Text, Image }` + `image_data: Option<Vec<u8>>`
- v1 去重使用 **PNG bytes 的 SHA256**（避免解码开销，v1 够用；像素级去重作为未来优化）

**消息类型：** `src-tauri/src/network/message.rs`
- 新增 `ClipboardImage { device_id, image_data: String (base64 PNG), thumbnail: Option<String> (base64 PNG), hash, timestamp }`
- `image_data` 和 `thumbnail` 用 `String` 而非 `Vec<u8>`，JSON 序列化自然编码为 base64 字符串
- 保持 `ClipboardText` 不变，向后兼容
- 图片大小限制：**原图 5MB**（webp/raw），base64 编码后约 6.7MB，加上 JSON 开销控制在 8MB 以内

**剪贴板写入：** `src-tauri/src/clipboard/writer.rs`
- 新增 `write_image(data: Vec<u8>)` 方法
- 完整路径：读 BLOB → `image::load_from_memory(&png_bytes)` 解码为 RGBA → 构造 `arboard::ImageData { width, height, bytes: rgba_pixels }` → `clipboard.set_image()`
- 需要引入 `image` crate 解码 PNG

**消息分发：** `src-tauri/src/app.rs`
- `handle_outgoing` 分支增加图片类型的序列化和广播
- `handle_incoming` 分支增加图片类型处理：base64 解码 → 调用 `writer.write_image()` 写入系统剪贴板 + 调 storage 入历史

**存储：** `src-tauri/src/clipboard/storage.rs`
- 数据库迁移（schema version 检查）：
  ```sql
  ALTER TABLE clipboard_history ADD COLUMN content_type TEXT NOT NULL DEFAULT 'text';
  ALTER TABLE clipboard_history ADD COLUMN image_data BLOB;
  ALTER TABLE clipboard_history ADD COLUMN thumbnail BLOB;
  CREATE INDEX idx_content_type ON clipboard_history(content_type);
  ```
- 缩略图：接收端生成。`image::load_from_memory()` 解码 PNG → `image.thumbnail(200, 200)`（最长边 200px，保持宽高比）→ PNG 编码存入 `thumbnail BLOB`
- 查询时根据 `content_type` 返回不同格式（文本返回 `content`，图片返回 `image_data` + `thumbnail`）

**前端：** `ui/index.html`
- 图片条目用 `<img src="data:image/png;base64,...">` 展示缩略图
- 分页加载（每页 50 条），Intersection Observer 懒加载缩略图
- 点击缩略图放大查看
- 点击"复制到剪贴板"触发 `copy_image_to_clipboard` IPC

**IPC 命令新增：**
- `copy_image_to_clipboard { id: u64 }`：从 storage 读取图片数据，解码为 RGBA，写入系统剪贴板
- `clear_history`（F1 已有）

**新增 crate：**
```toml
image = { version = "0.25", default-features = false, features = ["png"] }
```

**改动文件：** `monitor.rs`, `writer.rs`, `storage.rs`, `message.rs`, `app.rs`, `lib.rs`, `index.html`, `Cargo.toml`

## Open Questions

1. **⚠ 阻塞问题：Tauri v2 dock 隐藏 API** — `tray-icon` feature 是否提供 `set_activation_policy` 或等价 API？如果 Tauri 不暴露此 API，走 `objc2` 需要 unsafe 代码（违反 CLAUDE.md）。**必须在编码前确认。**
2. **arboard 图片格式：** `arboard::ImageData` 接受什么像素格式（RGBA/BGRA），当前版本是否稳定支持跨平台图片操作？需在编码前写测试验证。
3. **缩略图生成位置：** 已决定接收端生成——先传输原图，接收端解码后生成缩略图。无需再讨论。
4. **图片去重策略：** 已决定 v1 用 PNG bytes SHA256，简单够用。像素级去重留到未来。

## Success Criteria

- 清空历史：API 调用后历史列表清空，UI 即时反映
- Dock 隐藏：启动后 macOS Dock 不显示图标，Windows/Linux 无影响
- 双图标修复：启动后系统托盘只有一个图标
- 安全配对：陌生设备连接时弹出确认对话框，接受后永久放行
- 图片同步：复制图片 → 另一设备历史中出现缩略图 → 点击恢复到系统剪贴板
